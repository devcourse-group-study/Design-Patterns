# State Pattern 

> 객체 내부의 상태 변경에 따라 객체의 행동이 달라지는 패턴

## 특징 
상태에 특화된 행동들을 분리해낼 수 있으며, 새로운 행동을 추가하더라도 다른 행동에 영향을 주지 않는다.

## 장점 및 단점
### 장점 
- 상태에 따른 동작을 개별 클래스로 옮겨 관리가 가능함.
- 기존의 특정상태에 따른 동작을 변경하지 않고 새로운 상태의 동작을 추가할 수 있음.
- 코드의 복잡도를 줄일 수 있음.

### 단점 
- 복잡도가 증가.
